Getting Started with DevOps

Prerequisites

Before starting your DevOps journey, you'll need to be familiar with these essential foundations:

1. Basic Command Line Skills

You'll need to be comfortable with basic terminal commands:


# Navigate directories
cd /path/to/directory    # Change directory
ls                       # List files
pwd                      # Print working directory

# File operations
mkdir new-directory     # Create directory
touch newfile.txt       # Create file
cp file1 file2         # Copy files
mv file1 file2         # Move/rename files

2. Version Control Basics (Git)

Essential Git commands for beginners:


# Basic Git workflow
git init               # Start a new repository
git clone <url>        # Clone existing repository
git add .             # Stage changes
git commit -m "message" # Commit changes
git push              # Upload to remote
git pull              # Download from remote

3. Basic Programming Concepts

Understanding these programming concepts is helpful:

  • Variables and data types
  • Control structures (if/else, loops)
  • Functions and modules
  • Error handling
  • Basic algorithms and data structures

4. Important Tools to Install


# For Ubuntu/Debian
sudo apt update
sudo apt install -y git
sudo apt install -y docker.io
sudo apt install -y python3-pip

5. Learning Path Recommendation

  1. Week 1-2: Command Line & Git
  2. Week 3-4: Docker Basics
  3. Week 5-6: Basic Python Scripting
  4. Week 7-8: Introduction to Cloud Concepts
  5. Week 9-10: Basic CI/CD Concepts

Quick Start Guide for DevOps Beginners

Your First DevOps Project

1. Setup Your Development Environment


# Create a project directory
mkdir my-first-devops-project
cd my-first-devops-project

# Initialize git
git init

# Create a simple Python application
touch app.py
touch Dockerfile
touch requirements.txt

2. Create a Simple Python Web Application

In app.py:


from flask import Flask
app = Flask(__name__)

@app.route('/')
def hello():
    return 'Hello, DevOps!'

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000)

In requirements.txt:


flask==2.0.1

3. Create Your First Dockerfile


FROM python:3.9-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY app.py .
CMD ["python", "app.py"]

4. Build and Run Your First Container


# Build the Docker image
docker build -t my-first-app .

# Run the container
docker run -p 5000:5000 my-first-app

5. Basic Git Operations


# Create .gitignore
echo "__pycache__" > .gitignore
echo "*.pyc" >> .gitignore

# Commit your code
git add .
git commit -m "My first DevOps project"

DevOps Architecture for Beginners

Basic DevOps Architecture

1. Components Overview

graph TD A[Developer] -->|Git Push| B[Source Control] B -->|Trigger| C[CI/CD Pipeline] C -->|Build| D[Container Image] D -->|Deploy| E[Kubernetes Cluster] E -->|Monitor| F[Monitoring Tools]

2. Essential Components Explained

Version Control (Git)
  • Stores your code
  • Tracks changes
  • Enables collaboration
Continuous Integration (CI)
  • Automatically builds code
  • Runs tests
  • Reports issues
Continuous Deployment (CD)
  • Automates deployment
  • Ensures consistency
  • Reduces human error
Containers (Docker)
  • Package applications
  • Ensure consistency
  • Easy deployment
Orchestration (Kubernetes)
  • Manages containers
  • Handles scaling
  • Ensures availability

3. Simple Architecture Example

Let's create a basic web application setup:

  web:
    build: .
    ports:
      - "5000:5000"

Kubernetes Best Practices for Beginners

Kubernetes Basics

1. Your First Pod

Create a simple pod running nginx:


# my-first-pod.yaml
apiVersion: v1
kind: Pod
metadata:
  name: my-nginx
spec:
  containers:
  - name: nginx
    image: nginx:latest
    ports:
    - containerPort: 80
            

kubectl apply -f my-first-pod.yaml
kubectl get pods

2. Basic Commands

Essential kubectl commands:


# View resources
kubectl get pods
kubectl get services
kubectl get deployments

# Get details
kubectl describe pod <pod-name>

# View logs
kubectl logs <pod-name>

# Execute commands in pod
kubectl exec -it <pod-name> -- /bin/bash

3. Creating a Deployment


# my-first-deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: nginx
        image: nginx:latest
        ports:
        - containerPort: 80

4. Common Mistakes to Avoid

  • Not setting resource limits
  • Using latest tag
  • Not having health checks
  • Forgetting labels

5. Simple Service Example


# my-first-service.yaml
apiVersion: v1
kind: Service
metadata:
  name: my-service
spec:
  selector:
    app: my-app
  ports:
  - port: 80
    targetPort: 80
  type: LoadBalancer

CI/CD Pipeline Basics for Beginners

Getting Started with CI/CD

1. GitHub Actions Example

Create your first workflow:


# .github/workflows/main.yml
name: My First Pipeline
on: [push]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2

      - name: Set up Python
        uses: actions/setup-python@v2
        with:
          python-version: '3.9'

      - name: Install dependencies
        run: |
          python -m pip install --upgrade pip
          pip install -r requirements.txt

      - name: Run tests
        run: python -m pytest

      - name: Build Docker image
        run: docker build -t my-app .

2. Understanding Pipeline Stages

  1. Source: Get code from repository
  2. Build: Compile/package code
  3. Test: Run automated tests
  4. Deploy: Push to production

3. Basic Jenkins Pipeline


// Jenkinsfile
pipeline {
    agent any

    stages {
        stage('Build') {
            steps {
                sh 'pip install -r requirements.txt'
            }
        }

        stage('Test') {
            steps {
                sh 'python -m pytest'
            }
        }

        stage('Deploy') {
            steps {
                sh 'docker build -t my-app .'
            }
        }
    }
}

4. Testing in CI/CD


# test_app.py
def test_homepage():
    response = client.get('/')
    assert response.status_code == 200
    assert b"Hello, DevOps!" in response.data

5. Deployment Strategies

  1. Basic Deploy: Direct update
  2. Blue-Green: Two identical environments
  3. Canary: Gradual rollout
  4. Rolling: Update in batches

Getting Started with Prometheus Monitoring

Basic Prometheus Setup

1. Simple Prometheus Configuration


# prometheus.yml
global:
  scrape_interval: 15s

scrape_configs:
  - job_name: 'my-app'
    static_configs:
      - targets: ['localhost:5000']

2. Adding Metrics to Your App


from prometheus_client import Counter, Histogram
from flask import Flask

app = Flask(__name__)
requests_total = Counter('requests_total', 'Total requests')
request_duration = Histogram('request_duration_seconds', 'Request duration')

@app.route('/')
@request_duration.time()
def hello():
    requests_total.inc()
    return 'Hello, DevOps!'

3. Basic Queries


# Request rate
rate(requests_total[5m])

# 95th percentile latency
histogram_quantile(0.95, rate(request_duration_seconds_bucket[5m]))
